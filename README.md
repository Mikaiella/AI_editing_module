**Модуль автоматизованого редагування** — приклад інтелектуального інструменту допомоги авторам у межах вбудованого редактора літературної веб-платформи.
Він створений з метою оптимізації процесу ручного редагування на основі етичного та контрольованого використання штучного інтелекту.

🚩 **Модуль не виконує автоматичного переписування тексту та не замінює автора**.

Репозиторій містить **демонстраційну sandbox-версію** інтелектуального модуля автоматизованого редагування текстів. На відміну від повної реалізації, інтегрованої до літературної платформи **Dragon Tale**, у цій версії немає додаткових механізмів перевірки, бази даних та інших компонентів платформи. Вона призначена виключно для показу роботи та основних механізмів модуля.

## 💫 Модуль у редакторі Dragon Tale

**Повноцінну роботу модуля** можна протестувати в межах **[Dragon Tale](https://dragon-tale.vercel.app/)**.
Для цього необхідно:

1. Зареєструватися у системі.
2. Створити твір.
3. Додати до цього твору главу.
4. Перейти до режиму редагування глави.
5. Виділити коректний текстовий фрагмент.
6. Почати роботу з модулем.

## ✨ Основні особливості модуля

- Контрольована взаємодія з LLM.
- Попередня валідація обраного текстового фрагмента.
- Обов'язкове надання згоди на передачу тексту сторонньому сервісу для ШІ-редагування.
- Використання готової бібліотеки шаблонів із структурованими запитами, розробленими спеціально для роботи з художньою прозою.
- Повний контроль користувача над власним текстом.
- Визначення факту застосування ШІ-відредагованого текстового фрагменту в фінальному тексті глави при збереженні (дворівнева перевірка, зокрема з використанням методу шинглів).
- Демонстрація факту використання ШІ-редагування.

## 🛠️ Використані технології

- **Full-stack Next.js з TypeScript** — для реалізації демонстраційного full-stack застосунку.
- **React** — для побудови користувацького інтерфейсу.
- **Bootstrap 5 та React-Bootstrap** — для стилізації та використання готових UI-компонентів.
- **OpenAI SDK** — для інтеграції з Groq API та роботи з моделлю Llama-3.1-8b-instant.
- **Vitest** — для модульного тестування механізмів попередньої валідації та складових алгоритму визначення використання ШІ-редагування.

## 💻 Запуск у локальному середовищі

🔑 **Для роботи модуля обов'язковою є наявність ключа для взаємодії з Groq API (отримати його можна на [Groq Console](https://console.groq.com/home)).**

### 📋 Інструкція запуску:

1. Клонуйте цей репозиторій:

```
git clone https://github.com/Mikaiella/AI_editing_module.git
```

2. Перейдіть до фактичної папки модуля:

```
cd demo_module
```

3. Встановіть залежності:

```
npm install
```

4. Створіть файл _.env.local_ (можна скопіювати з _.env.example_) та додайте до нього отриманий API-ключ:

```
GROQ_API_KEY=KEY_GOES_HERE
```

5. Запустіть проєкт:

```
npm run dev
```

6. Відкрийте посилання:

```
http://localhost:3000/
```

📝 У проєкті вже наявні дані демонстраційної глави (файл _demo-chapter.json_), які включають текст та стан визначення використання ШІ-редагування, що за потреби можуть бути змінені.

## ⚡️ Сценарій використання модуля

1. Виділіть текстовий фрагмент у редакторі.
2. Натисніть кнопку активації модуля.
3. Підтвердьте згоду на передачу тексту для ШІ-редагування.
4. Оберіть шаблон редагування.
5. Порівняйте оригінальний та ШІ-відредагований варіанти.
6. Застосуйте обраний варіант.
7. Збережіть зміни (алгоритм автоматично визначить використання ШІ-редагування).
